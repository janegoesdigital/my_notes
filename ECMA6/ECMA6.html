Functions and the arrow thing

What

Replaces the palaver of using the word function everywhere with nice little arrow functions. The function keyword is replaced with an arrow =>. Very nice when you are passing a function to another function ie a callback (I think)

Compare:

var num = [1,2,3];
num.map(function(x) {
return X * 2;
})

with

num.map((x) => x * 2);

ES6 has a “thin arrow” and “fat arrow” syntax. You do not actually have to return explicitly. Just the last statement in that function is what gets returned – that last expression. These two bits of code are achieving the same thing but the second is a much nicer syntax.

Need curly brackets when the functions are across lines

When the arrow function is on one line, you do not need the curly braces while mapping the example. If the function is split the arrow for multiple lines, you do need those braces.
